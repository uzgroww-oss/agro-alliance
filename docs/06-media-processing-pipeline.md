# Media Processing Pipeline

## Agro Alliance — File Upload, Optimization, Storage & CDN Delivery

---

## 1. Overview

Media Processing handles all file uploads (images, videos, documents) from user uploads (blogger photos, video thumbnails, article images, file attachments). The pipeline processes, optimizes, stores in Cloudflare R2, and delivers via CDN.

---

## 2. Architecture

```
┌──────────┐    ┌───────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│  Upload  │ → │  Validate │ → │ Optimize │ → │   Store  │ → │   CDN    │
│  Client  │    │  Sanitize │    │  Resize  │    │ R2 Bucket│    │ Delivery │
└──────────┘    └───────────┘    └──────────┘    └──────────┘    └──────────┘
     │               │               │               │               │
     ▼               ▼               ▼               ▼               ▼
Edge Function    Virus Scan      Sharp/WASM     Cloudflare R2    Cloudflare CDN
                 Magic Bytes     WebP Convert   Public/Private   Cache-Control
```

## 3. Upload Endpoints

### 3.1 Blogger Photo Upload
**Endpoint**: `POST /api/me/profile/photo` (future — currently uses PUT /api/me/profile with base64)
**Method**: Multipart form data
**Max Size**: 5MB
**Accepted Types**: image/webp, image/jpeg, image/png

### 3.2 Article Image Upload (Admin/Future)
**Endpoint**: `POST /api/news/images`
**Max Size**: 10MB
**Accepted Types**: image/webp, image/jpeg, image/png, image/gif

### 3.3 Contact Form Attachment (Future)
**Endpoint**: Included in `POST /api/contact` as base64
**Max Size**: 10MB
**Accepted Types**: image/*, application/pdf, text/plain

## 4. Validation & Sanitization

| Check | Method | Action |
|-------|--------|--------|
| File type | Magic bytes verification | Reject if mismatch with extension |
| File size | Compare against max | Reject with "Fayl hajmi juda katta" |
| Image dimensions | Read header | Reject if < 50px or > 8000px |
| Virus/malware | ClamAV (future) | Quarantine if infected |
| Filename sanitization | Strip special chars, UUID rename | Replace with safe UUID |
| Duplicate check | MD5 hash of content | Skip if identical to existing |

## 5. Optimization Pipeline

### 5.1 Image Processing (Sharp / WASM-based)

| Variant | Size | Quality | Format | Use Case |
|---------|------|---------|--------|----------|
| thumbnail | 150x150 | 80% | WebP | List cards, sidebar, tables |
| small | 320x240 | 85% | WebP | Grid cards, blog cards |
| medium | 640x480 | 85% | WebP | Featured sections, profile cover |
| large | 1200x800 | 90% | WebP | Article hero, lightbox |
| original | as-is | as-is | as-is | Preserved for editing |

### 5.2 Processing Steps
```
1. Decode → WebP format target
2. Strip EXIF/metadata (privacy + size)
3. Resize to target dimensions (maintain aspect ratio)
4. Auto-orient based on EXIF rotation
5. Generate blurred placeholder (base64, 20px)
6. Generate thumbnail variant
7. Compress to target quality
8. Store all variants
```

### 5.3 File Naming Convention
```
/users/{user_id}/{entity}/{uuid}-{variant}.webp

Examples:
/users/abc123/profile/550e8400-thumbnail.webp
/users/abc123/profile/550e8400-medium.webp
/users/abc123/videos/550e8400-small.webp
/news/2024/05/dronlar-ekin-kuzatish-medium.webp
/contact/attachments/550e8400-original.pdf
```

### 5.4 Blurhash / Low-Quality Image Placeholders (LQIP)
- Generate 20px-wide blurred base64 variant on upload
- Store in database alongside image URL
- Frontend shows blurred placeholder → loads full image on reveal
- Used in: news listing, blogger cards, video thumbnails

## 6. Cloudflare R2 Storage

### 6.1 Bucket Structure

```
Bucket: agro-alliance-media
├── public/           # Publicly accessible via CDN
│   ├── bloggers/     # Blogger profile photos + covers
│   ├── news/         # News article images
│   ├── partners/     # Partner logos
│   ├── site/         # Site assets (logos, backgrounds)
│   └── uploads/      # General uploads
├── private/          # Signed URL access only
│   ├── contracts/    # Partner contracts (PDF)
│   ├── attachments/  # Contact form file attachments
│   └── temp/         # Temporary processing workspace
└── backups/          # Database backups, logs
```

### 6.2 Access Control

| Bucket | Visibility | TTL | Cache-Control |
|--------|------------|-----|---------------|
| public/* | Public read | 1 year | `public, max-age=31536000, immutable` |
| private/* | Signed URL only | 60s | `private, no-cache` |
| temp/* | Uploader only | 15 min | `private, no-cache` |

### 6.3 Signed URLs
- Generated by Edge Functions using R2 presigned URL API
- Default TTL: 60 seconds
- Used for: contract downloads, attachment access, temp file access
- Path format: `{bucket}.r2.dev/private/contracts/{uuid}.pdf?signature={sig}&expiry={ts}`

### 6.4 CDN Configuration
- **Provider**: Cloudflare (R2 is served through Cloudflare's global network)
- **Caching**: Cache everything in `public/` at edge for 1 year
- **Cache Invalidation**: On image replacement, use new filename (cache-bust via UUID)
- **Optimization**: Cloudflare Polish (lossy image compression) — optional
- **WebP support**: Cloudflare automatically serves WebP to compatible browsers

## 7. Edge Function Upload Handlers

### 7.1 Photo Upload Handler
```
POST /api/me/profile/photo
Content-Type: multipart/form-data

Process:
1. Validate file (type, size, dimensions)
2. Generate UUID filename
3. Process variants: thumbnail (150x150), medium (640x640)
4. Upload to R2: public/bloggers/{userId}/profile/{uuid}-{variant}.webp
5. Update profiles.photo_url with medium variant URL
6. Return { photo_url: "https://cdn.agroalliance.uz/...medium.webp" }
```

### 7.2 Video Thumbnail Proxy
```
No direct upload — thumbnails are fetched from platform APIs (YouTube, TikTok)

Process:
1. When video link is added, platform API returns thumbnail URL
2. Edge Function fetches thumbnail from source
3. Processes variants (thumbnail, small, medium)
4. Uploads to R2: public/bloggers/{userId}/videos/{uuid}-{variant}.webp
5. Updates videos.thumbnail with medium variant URL
```

### 7.3 News Image Upload (Future)
```
POST /api/news/images
Content-Type: multipart/form-data

Process:
1. Validate file
2. Generate UUID filename
3. Process all 5 variants
4. Upload to R2: public/news/{year}/{month}/{uuid}-{variant}.webp
5. Return URLs for all variants
```

## 8. Image URL Resolver

Frontend should use a consistent URL pattern:

```
// R2 Direct (no CDN proxy):
https://pub-{bucket-id}.r2.dev/public/{path}

// With Custom Domain (CDN):
https://media.agroalliance.uz/{path}

// Variant selection by context:
function getImageUrl(base: string, variant: 'thumb' | 'small' | 'medium' | 'large'): string {
  return base.replace(/-original\./, `-${variant}.`);
}
```

## 9. Background Processing

### 9.1 Upload Queue
- Large uploads (>2MB) processed asynchronously via queue
- Small uploads processed inline for immediate response
- Queue: PostgreSQL + pgmq (future: Redis/BullMQ)

### 9.2 Cleanup Job (Daily Cron)
- Delete temp files older than 24 hours from R2 `temp/` bucket
- Delete unused variants when original is removed
- Archive logs older than 90 days

### 9.3 Re-optimization Job (Weekly Cron)
- Re-process all images with updated optimization settings
- Useful when compression quality settings change
- Only processes images older than 30 days (recent ones are likely fine)

## 11. Security

| Concern | Mitigation |
|---------|------------|
| Malicious upload (exe, script) | Magic byte verification + extension whitelist |
| Large file DOS | Size limit + multipart streaming |
| Path traversal | UUID filenames only, never user-provided paths |
| Hotlinking | (Optional) Referrer-based access control on CDN |
| Exif data leakage | Strip EXIF on all uploads |
| Duplicate abuse | MD5 hash dedup before processing |
| Rate limiting | 10 uploads/hour per user |

## 12. Frontend Integration

### 12.1 Current State (Base64)
- Currently, blogger photo upload sends base64-encoded image in JSON body
- **Issue**: Base64 is ~33% larger than binary, no optimization, no CDN
- **Migration**: Change to multipart/form-data upload to separate endpoint

### 12.2 Target State (Direct Upload)
```
1. Frontend: File picker → POST /api/me/profile/photo (multipart)
2. Backend: Edge Function processes and stores
3. Response: { photo_url: "https://media.agroalliance.uz/...medium.webp" }
4. Frontend: Update image src immediately
```

### 12.3 Upload Component UX
- Drag-and-drop support
- Preview before upload

- Upload progress bar

- Maximum 10MB total per upload session
